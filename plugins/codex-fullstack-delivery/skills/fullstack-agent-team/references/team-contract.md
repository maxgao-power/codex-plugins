# 全栈 Agent 团队参考索引

使用 `fullstack-agent-team` skill 时，先遵守 `SKILL.md` 的通用强规则和门禁。需要细节时按角色加载本目录下的参考文件，不要一次性读取所有文件。

## 按角色加载

- 技术负责人：读取 `technical-lead.md`，用于项目发现、验收单、调度、集成循环、Codex 浏览器演示、中断恢复和最终总结。
- 后端 Agent：读取 `backend-agent.md`，用于 DDD 触发判断、接口契约、后端检查清单、后端任务包和后端交接。
- 前端 Agent：读取 `frontend-agent.md`，用于接口消费、前端可测性要求、前端任务包和前端交接。
- 测试 Agent：读取 `test-agent.md`，用于测试用例设计、自动化映射、测试流程、验证纪律和测试任务包。

## 加载原则

- 主会话默认只读取技术负责人需要的章节；不要把所有 Agent 细则加载进主上下文。
- 给某个 Agent 下发任务时，只附对应角色文件中的任务包要求和必要路径。
- 接口契约、测试用例、演示结果和交接摘要使用文件路径或短摘要传递；长日志留在工具输出或责任 Agent 上下文。
- 若角色边界不清，先回到 `SKILL.md` 的核心边界和强制门禁裁决。
