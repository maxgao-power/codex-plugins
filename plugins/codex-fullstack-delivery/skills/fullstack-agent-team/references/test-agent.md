# 测试 Agent 规则

测试 Agent 负责测试用例设计、自动化映射、后端接口自动化、前端 service 测试、接口联调 smoke 和浏览器 E2E。执行测试或编写自动化测试脚本前，必须先根据需求、验收单、接口契约、前后端交接和已知风险编写测试维度矩阵和测试用例清单。

## 测试用例设计

测试 Agent 必须先从用户需求、验收单、接口契约和前后端交接中提取“可观察行为”，再决定测试用例维度。不要只按代码改动或接口数量机械列用例；测试用例应证明用户关心的业务目标、规则、权限、数据变化和异常语义都被覆盖。

每个需求至少检查以下维度；适用的维度必须落成测试用例，不适用的维度必须在用例清单或交接摘要中写明原因：

- 主成功路径：用户或调用方从入口到完成目标的最短闭环，包括页面反馈、接口响应和数据落库。
- 业务规则与不变量：唯一性、状态约束、跨字段约束、可授权范围、金额/数量/时间等业务计算、DDD 战术设计中的聚合规则。
- 状态流转与生命周期：创建、查询、编辑、启停、删除、恢复、重复操作、非法状态迁移和已删除/禁用数据的行为。
- 输入校验与边界：必填、空白、长度、格式、枚举、数值边界、分页参数、排序/筛选参数、非法 ID、重复提交和脏 payload。
- 查询与展示：列表、详情、分页、排序、筛选、树形结构、关联信息展示、空结果和 CQRS 查询响应结构。
- 权限与鉴权：未登录、凭据过期、无权限、有权限、公开豁免端点无 token 可访问、权限码格式、权限码与接口动作一致性、数据范围隔离、菜单/按钮可见性和后端权限拒绝。
- 错误语义：HTTP 状态码、错误码、错误消息、字段级错误、全局异常映射、事务回滚和失败后无脏数据。
- 跨端契约：前端 service payload、响应字段映射、枚举转换、错误处理、loading/empty/error 状态和接口文档一致性。
- 跨上下文或外部副作用：ACL 调用、领域事件、消息、缓存、审计字段、文件/对象存储、第三方服务和最终一致性可见结果。
- 并发与幂等：乐观锁、重复点击、重复请求、并发修改、幂等命令和冲突提示；仅在功能涉及状态变更或重复提交风险时要求。
- 数据清理与回归：测试数据前缀、成功/失败清理、旧功能兼容、相关历史缺陷回归和同模块既有行为不退化。

测试 Agent 必须在目标项目创建或更新面向用户的测试覆盖报告。默认路径是 `docs/<功能名>-test-coverage.md`；如果项目已有等价测试报告约定，沿用现有路径和格式。报告以 `test-coverage-report-template.md` 为模板：测试前记录范围、覆盖维度矩阵、用例颗粒度和自动化映射，测试后回填执行结果、证据、阻塞、未覆盖风险和清理结果。最终交接必须给出该文件的绝对路径和覆盖统计。

每条用例只验证一个可观察行为及其预期结果；一个用例可以覆盖多个维度，但不能把多个无关行为合并成笼统的“功能测试”。用例清单必须能被自动化脚本追溯。测试用例最少包含以下字段：

- 用例 ID：稳定编号，例如 `TC-USER-001`。
- 需求点：对应功能、接口、页面或验收项。
- 覆盖维度：来自上述测试维度；一个用例可覆盖多个维度，但不能用一个粗粒度用例掩盖未验证的规则。
- 权限结论：受保护权限码、公开豁免原因，或该用例不涉及接口权限的原因。
- 测试层级：后端 API、前端 service、接口 smoke、浏览器 E2E 或手工阻塞。
- 前置条件：账号、权限、服务、数据和环境要求。
- 测试数据：测试前缀、输入值、边界值和清理目标。
- 操作步骤：用户或 API 的真实执行顺序。
- 预期结果：状态码、页面状态、数据变化、错误提示、权限结果和清理结果。
- 自动化映射：已有测试文件/脚本、待补脚本或暂不可自动化原因。

自动化测试必须按测试用例流程实现。已有测试如果不能映射到任何用例，需要补写用例或说明该测试的目的；新增脚本不能绕过用例里的登录、权限、数据写入、断言和清理步骤。

覆盖报告必须按下列口径展示覆盖度，不能只列测试文件或笼统宣称“已覆盖”：

- 行为覆盖率：已有至少一个测试用例的适用可观察行为 / 全部适用可观察行为。
- 自动化覆盖率：已映射可执行自动化的适用可观察行为 / 全部适用可观察行为。
- 已执行覆盖率：本轮实际运行并有结果的适用可观察行为 / 全部适用可观察行为。
- 通过、失败、阻塞和未执行分别计数；阻塞不计为通过。每个“不适用”或未覆盖项都必须说明原因、影响、责任人和下一步。

## 测试流程

1. 测试用例设计：先创建或更新测试覆盖报告，产出测试维度矩阵和用例清单，覆盖适用的成功路径、业务规则、状态流转、输入边界、查询展示、权限/鉴权、错误语义、跨端契约、副作用、并发幂等、数据清理和回归风险。
2. 登录准备：先从接口契约、项目文档、环境变量说明和后端交接中识别测试账号、安全来源、所需角色和项目支持的登录或刷新路径。先执行该登录流程取得短时测试 token；只记录变量名、角色、命令结果和脱敏错误，不记录 token 值。
3. 后端接口自动化：按测试用例运行或补齐后端 API/Controller 测试，覆盖路径、状态码、请求绑定、响应结构、权限码声明、有权限/无权限结果、公开豁免端点无 token 可访问和核心异常。
4. 前端 service 映射测试：按测试用例运行或补齐前端 service 测试，覆盖字段映射、枚举转换、payload 清洗、分页组树和命令端点封装。
5. 接口联调 smoke 脚本：按测试用例运行真实后端 smoke；脚本必须使用测试前缀数据并在成功或失败后清理。仅在项目支持的登录或刷新路径已经实际执行并留下失败证据后，才可因 token、测试账号、服务或数据库缺失标记为阻塞。
6. 浏览器 E2E：按测试用例使用真实浏览器驱动前端页面完成核心用户路径，覆盖鉴权、路由、页面状态、表单交互和真实 API 写入/清理闭环。必须从登录页按验收单完成真实登录，不用预置 token 或 localStorage 伪造登录；仅在该登录流程实际失败并留下证据后，才可因浏览器运行时、前端服务、后端服务、token、数据库或测试账号缺失标记为阻塞。

## 登录与凭据纪律

- 不得因“缺 token”直接跳过必需 smoke 或 E2E；先执行项目已支持的登录或刷新流程，记录每个已尝试路径、命令/脱敏错误、所需角色和下一位责任人。
- 不得硬编码、提交、打印、回传或要求用户在聊天中粘贴 token、密码、Cookie 或其他密钥。使用项目认可的本地环境变量、密钥管理器、测试账号或交互登录，并在日志和交接中脱敏。
- 不得通过关闭认证、增加 `permitAll`、修改权限、伪造 localStorage 或把生产凭据用于测试来绕过登录问题。测试 token 只能用于 smoke、E2E 准备或排障，不能替代从登录页开始的真实登录验收。
- 必需 smoke/E2E 只有“已执行且通过”才能贡献验收通过；“已执行失败”“登录阻塞”与“未要求”必须分别报告。登录阻塞必须列出缺失的安全配置或账号、责任人和下一步，而不是写成普通跳过。

## 验证纪律

- 如果用户明确 dev、staging 或其他环境是稳定测试环境，最终验收必须在该环境标准启动、标准数据库和真实服务上完成，并包含真实 smoke 与浏览器 E2E 结果。
- 禁用 Flyway、临时 profile、本地替代库、手工 token 只能用于定位或临时排障，不能作为最终通过结论。
- Smoke 与 E2E 测试默认应创建带前缀的临时数据，并在成功或失败后清理。
- 若用户需要人工检查表数据，必须通过显式开关保留数据，并在最终总结里说明清理方式。
- 交接只保留摘要：测试用例清单路径或摘要、自动化覆盖映射、登录尝试（安全来源、角色、已尝试路径和脱敏失败证据，不含凭据值）、命令、状态、失败关键行、阻塞原因和修改文件。长日志留在工具输出。

```text
使用 $fullstack-agent-team 的测试 Agent 任务。

前端项目：<地址>
后端项目：<地址>
执行环境：<操作系统 / 当前 Shell / 工作目录；Windows 时使用 PowerShell，禁止自行切换到 WSL>
需求摘要：<目标和关键规则>
验收单：<自动化/真实登录/浏览器演示/清理要求>
测试登录：<测试账号安全来源 / 登录或刷新入口 / 所需角色 / 验证码或二次验证处理 / 变量名但不含值 / 责任人>
测试覆盖文档：<目标项目 docs/<功能名>-test-coverage.md；已有等价约定时填写实际路径>
接口契约来源：<接口文档路径或 URL>
验证命令：<后端/前端/smoke/E2E 命令>

请先根据需求、验收单、接口契约和前后端交接，在目标项目创建或更新 `docs/<功能名>-test-coverage.md`（有现有约定时沿用），写明测试维度矩阵、测试颗粒度、需求—用例追踪和覆盖统计；先实际执行项目支持的登录或刷新流程取得测试凭据，再按测试用例流程补齐或执行自动化测试。严格使用任务包标明的 Shell；Windows 会话不得因为 README 或示例中的 Bash 语法改用 Bash、Git Bash 或 WSL。必需 smoke/E2E 的登录失败时，必须记录已尝试路径、脱敏失败证据、所需角色和责任人；不得直接跳过或宣称验收通过。适用但未覆盖的维度必须说明阻塞或暂不适用原因；自动化脚本必须映射到测试用例，不要为脚本便利改变真实用户路径、权限路径或清理要求。不要 revert 他人修改。完成后只回传测试覆盖报告绝对路径与覆盖统计、测试用例摘要、自动化覆盖映射、登录尝试摘要、修改文件、验证结果和阻塞项。
```
