# 测试 Agent 规则

测试 Agent 负责测试用例设计、自动化映射、后端接口自动化、前端 service 测试、接口联调 smoke 和浏览器 E2E。执行测试或编写自动化测试脚本前，必须先根据需求、验收单、接口契约、前后端交接和已知风险编写测试维度矩阵和测试用例清单。

## 测试用例设计

测试 Agent 必须先从用户需求、验收单、接口契约和前后端交接中提取“可观察行为”，再决定测试用例维度。不要只按代码改动或接口数量机械列用例；测试用例应证明用户关心的业务目标、规则、权限、数据变化和异常语义都被覆盖。

每个需求至少检查以下维度；适用的维度必须落成测试用例，不适用的维度必须在用例清单或交接摘要中写明原因：

- 主成功路径：用户或调用方从入口到完成目标的最短闭环，包括页面反馈、接口响应和数据落库。
- 业务规则与不变量：唯一性、状态约束、跨字段约束、可授权范围、金额/数量/时间等业务计算、DDD 战术设计中的聚合规则。
- 状态流转与生命周期：创建、查询、编辑、启停、删除、恢复、重复操作、非法状态迁移和已删除/禁用数据的行为。
- 输入校验与边界：必填、空白、长度、格式、枚举、数值边界、分页参数、排序/筛选参数、非法 ID、重复提交和脏 payload。
- 查询与展示：列表、详情、分页、排序、筛选、树形结构、关联信息展示、空结果和 CQRS 查询响应结构。
- 权限与鉴权：未登录、凭据过期、无权限、有权限、公开豁免端点无 token 可访问、权限码格式、权限码与接口动作一致性、数据范围隔离、菜单/按钮可见性和后端权限拒绝。
- 错误语义：HTTP 状态码、错误码、错误消息、字段级错误、全局异常映射、事务回滚和失败后无脏数据。
- 跨端契约：前端 service payload、响应字段映射、枚举转换、错误处理、loading/empty/error 状态和接口文档一致性。
- 跨上下文或外部副作用：ACL 调用、领域事件、消息、缓存、审计字段、文件/对象存储、第三方服务和最终一致性可见结果。
- 并发与幂等：乐观锁、重复点击、重复请求、并发修改、幂等命令和冲突提示；仅在功能涉及状态变更或重复提交风险时要求。
- 数据清理与回归：测试数据前缀、成功/失败清理、旧功能兼容、相关历史缺陷回归和同模块既有行为不退化。

用例清单可以保存在测试项目文档、测试目录或交接摘要中，但必须能被自动化脚本追溯。测试用例最少包含以下字段：

- 用例 ID：稳定编号，例如 `TC-USER-001`。
- 需求点：对应功能、接口、页面或验收项。
- 覆盖维度：来自上述测试维度；一个用例可覆盖多个维度，但不能用一个粗粒度用例掩盖未验证的规则。
- 权限结论：受保护权限码、公开豁免原因，或该用例不涉及接口权限的原因。
- 测试层级：后端 API、前端 service、接口 smoke、浏览器 E2E 或手工阻塞。
- 前置条件：账号、权限、服务、数据和环境要求。
- 测试数据：测试前缀、输入值、边界值和清理目标。
- 操作步骤：用户或 API 的真实执行顺序。
- 预期结果：状态码、页面状态、数据变化、错误提示、权限结果和清理结果。
- 自动化映射：已有测试文件/脚本、待补脚本或暂不可自动化原因。

自动化测试必须按测试用例流程实现。已有测试如果不能映射到任何用例，需要补写用例或说明该测试的目的；新增脚本不能绕过用例里的登录、权限、数据写入、断言和清理步骤。

## 测试流程

1. 测试用例设计：先产出测试维度矩阵和用例清单，覆盖适用的成功路径、业务规则、状态流转、输入边界、查询展示、权限/鉴权、错误语义、跨端契约、副作用、并发幂等、数据清理和回归风险。
2. 后端接口自动化：按测试用例运行或补齐后端 API/Controller 测试，覆盖路径、状态码、请求绑定、响应结构、权限码声明、有权限/无权限结果、公开豁免端点无 token 可访问和核心异常。
3. 前端 service 映射测试：按测试用例运行或补齐前端 service 测试，覆盖字段映射、枚举转换、payload 清洗、分页组树和命令端点封装。
4. 接口联调 smoke 脚本：按测试用例运行真实后端 smoke；脚本必须使用测试前缀数据并在成功或失败后清理。缺 token、后端服务、数据库或测试账号时标记为阻塞。
5. 浏览器 E2E：按测试用例使用真实浏览器驱动前端页面完成核心用户路径，覆盖鉴权、路由、页面状态、表单交互和真实 API 写入/清理闭环。缺浏览器运行时、前端服务、后端服务、token、数据库或测试账号时标记为阻塞。E2E 的鉴权路径应符合验收单；除非目标就是 token 恢复，否则不要用 token 注入宣称登录流程通过。

## 验证纪律

- 如果用户明确 dev、staging 或其他环境是稳定测试环境，最终验收必须在该环境标准启动、标准数据库和真实服务上完成，并包含真实 smoke 与浏览器 E2E 结果。
- 禁用 Flyway、临时 profile、本地替代库、手工 token 只能用于定位或临时排障，不能作为最终通过结论。
- Smoke 与 E2E 测试默认应创建带前缀的临时数据，并在成功或失败后清理。
- 若用户需要人工检查表数据，必须通过显式开关保留数据，并在最终总结里说明清理方式。
- 交接只保留摘要：测试用例清单路径或摘要、自动化覆盖映射、命令、状态、失败关键行、阻塞原因和修改文件。长日志留在工具输出。
