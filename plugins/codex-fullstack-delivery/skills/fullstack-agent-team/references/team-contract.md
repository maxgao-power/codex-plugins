# 全栈 Agent 团队协作契约

使用 `fullstack-agent-team` skill 进行真实前后端开发对接时，读取本文件作为详细执行规则。

## 项目地址发现

开始编辑代码前，收集以下信息：

- 前端：本地路径或仓库 URL、分支、包管理器、开发命令、测试命令、构建命令、环境变量文件要求、API base URL 配置方式。
- 后端：本地路径或仓库 URL、分支、语言和框架、开发命令、测试命令、数据库迁移命令、接口文档位置、环境变量文件要求。

如果是本地路径，状态更新和最终总结中使用绝对路径。如果是仓库 URL，先确认仓库是否已经在本地检出，再决定是否需要克隆或向用户请求访问权限。若当前工作区或权限无法访问必要项目，明确说明阻塞并请求用户提供可访问路径、仓库或项目内容。

## 上下文最小化规则

技术负责人只在主上下文保留可执行摘要：项目地址、分支、关键命令、接口契约路径、Agent 分工、阻塞问题、验证结果。不要把大段源码、完整命令输出、完整接口文档或全量文件清单贴进主上下文。

Agent 任务包必须包含：项目路径、角色职责、主要写入范围、接口契约来源、验证命令、交接格式，以及“你不是独自在代码库里工作，不要 revert 他人修改”。让 Agent 自己读取所属项目的 `AGENTS.md`、README、源码和测试。

如果 Agent 需要额外信息，优先提供文件路径、搜索模式或接口文档路径。只有影响决策的短片段才放入主上下文。

前端 Agent 和后端 Agent 的任务包必须明确：实现代码前先读取所属项目的 `AGENTS.md` 并按其中规范开发、测试和交接。如果项目没有 `AGENTS.md`，再读取 README、CLAUDE.md 或其他本地约定文件。技术负责人只保留“已读取并遵循项目规范”的摘要，不把完整规范复制进主上下文。

## 接口文档来源选择

按以下优先级选择接口契约来源：

1. 现有 OpenAPI 文档：`openapi.yaml`、`openapi.yml`、`openapi.json`、Swagger 导出文件或已生成的 API docs 地址。
2. 框架原生 schema：FastAPI schema、NestJS Swagger 装饰器、Spring/OpenAPI 注解、ASP.NET OpenAPI、Rails route docs、Laravel API resource、GraphQL schema、tRPC router、Zod route schema 或类似类型化路由定义。
3. 前端已经使用的生成客户端包。
4. 后端 Agent 新建的接口契约文件。

创建新契约文件时，默认优先使用 OpenAPI 3.1 YAML；如果项目已有其他格式，则沿用项目格式。

## 并行调度规则

- 后端 Agent 拥有接口契约、后端实现、后端测试和后端运行说明。
- 前端 Agent 拥有页面、组件、状态、API client、前端测试和用户侧体验。
- 技术负责人可以另启独立 Explorer 回答互不重叠的代码问题，但不要让多个 Agent 重复读取或修改同一模块。
- 接口未知时，后端 Agent 先产出契约；前端 Agent 可并行梳理已有 API client、页面结构和加载/错误态，不得对未文档化字段落地调用。
- 每个 Agent 完成后只交付摘要、修改文件、验证命令和失败关键行；长日志留在工具输出中。

## 后端 Agent 检查清单

- 实现代码前，读取后端项目 `AGENTS.md` 并遵循其中架构、包结构、命名、测试、迁移和安全规范。
- 设计新接口前，先检查已有路由、控制器、服务和类型定义。
- 保持项目现有鉴权、权限、分页、错误结构和命名风格。
- 在交给前端 Agent 前，必须更新或创建接口文档。
- 为所有新增或变更接口提供示例。
- 实现后端代码和后端测试。
- 交接时说明变更接口、文档路径、服务启动命令和测试命令。

## 前端 Agent 检查清单

- 实现代码前，读取前端项目 `AGENTS.md` 并遵循其中目录、组件、样式、状态、请求、测试和构建规范。
- 实现对接前先阅读接口文档。
- 沿用项目已有 API client、base URL 配置、鉴权 token 处理和错误处理模式。
- 不要猜测未文档化的字段、路径或状态码。
- 如果契约不完整，暂停该接口对接，并向技术负责人报告缺失项。
- 实现 UI 行为、加载态、空状态、错误态和符合项目习惯的前端测试。
- 交接时说明修改的页面/组件、消费的 API、开发命令和测试命令。

## 测试 Agent 流程

前后端实现完成后，技术负责人应创建独立测试 Agent，任务包包含前后端项目地址、接口契约路径、可运行命令、凭据/环境变量要求和交接格式。测试 Agent 按以下顺序执行：

1. 后端接口自动化：运行或补齐后端 API/Controller 测试，覆盖路径、状态码、请求绑定、响应结构和核心异常。
2. 前端 service 映射测试：运行或补齐前端 service 测试，覆盖字段映射、枚举转换、payload 清洗、分页组树和命令端点封装。
3. 接口联调 smoke 脚本：运行真实后端 smoke；脚本必须使用测试前缀数据并在成功或失败后清理。缺 token、后端服务、数据库或测试账号时标记为阻塞。
4. 浏览器 E2E：使用真实浏览器驱动前端页面完成核心用户路径，覆盖鉴权、路由、页面状态、表单交互和真实 API 写入/清理闭环。缺浏览器运行时、前端服务、后端服务、token、数据库或测试账号时标记为阻塞。

如果用户明确 dev、staging 或其他环境是稳定测试环境，最终验收必须在该环境标准启动、标准数据库和真实服务上完成，并包含真实 smoke 与浏览器 E2E 结果。禁用 Flyway、临时 profile、本地替代库、手工 token 只能用于定位或临时排障，不能作为最终通过结论。

Smoke 与 E2E 测试默认应创建带前缀的临时数据，并在成功或失败后清理。因此测试通过后业务表没有残留临时数据是预期。若用户需要人工检查表数据，必须通过显式开关保留数据，并在最终总结里说明清理方式。

测试 Agent 交接只保留摘要：命令、状态、失败关键行、阻塞原因和修改文件。长日志留在工具输出。

## 技术负责人集成循环

前后端 Agent 完成后，按以下循环集成：

1. 阅读 Agent 交接摘要和修改文件列表，只对风险点做定向源码检查。
2. 对照接口文档检查前端 API 调用。
3. 运行后端测试。
4. 运行前端类型检查、构建或测试。
5. 只有在需要且对工作区安全时启动服务。
6. 可行时验证至少一个成功路径和一个错误路径。
7. 分派失败项：
   - 接口缺失、schema 不匹配、状态码不匹配、鉴权不匹配：分派给后端 Agent，除非前端没有按文档实现。
   - 客户端调用错误、UI 状态错误、请求 payload 不符合文档：分派给前端 Agent。
   - 后端启动失败、迁移失败、缺接口、数据库 schema 不一致：分派给后端 Agent 诊断并提出修复方案。
   - 前端 service 映射、payload、UI 调用或状态处理错误：分派给前端 Agent 修复。
   - 环境、凭据、种子数据或跨项目配置问题：由技术负责人先排查；涉及共享数据库、Flyway `repair`、`out-of-order`、清库或生产样环境写操作时，必须说明风险并获得用户确认。
8. 持续迭代，直到验证通过，或被缺失凭据、服务不可用、用户决策等真实问题阻塞。

## 交接格式

后端 Agent 任务包：

```text
使用 $fullstack-agent-team 的后端 Agent 任务。

后端项目：<地址>
功能目标：<目标>
职责边界：API、接口文档、后端实现、后端测试和运行说明。
接口契约来源：<已有文档路径/待创建文档位置/未知则先产出>
验证命令：<后端测试/构建/启动命令>

请先读取并遵循后端项目 AGENTS.md；如果没有，读取 README、CLAUDE.md 或其他项目约定。不要 revert 他人修改。完成后只回传交接摘要、修改文件、接口文档路径、验证结果和阻塞项。
```

前端 Agent 任务包：

```text
使用 $fullstack-agent-team 的前端 Agent 任务。

前端项目：<地址>
功能目标：<目标>
职责边界：页面、组件、状态管理、API client、前端测试和用户侧体验。
接口契约来源：<后端交付的接口文档路径或 URL>
验证命令：<前端测试/类型检查/构建/启动命令>

请先读取并遵循前端项目 AGENTS.md；如果没有，读取 README、CLAUDE.md 或其他项目约定。前端真实对接只能消费接口文档，不要猜字段、路径或错误结构。不要 revert 他人修改。完成后只回传交接摘要、修改文件、消费的接口、验证结果和阻塞项。
```

```text

验证结果：<通过 / 失败关键行>
备注：<鉴权/环境变量/迁移说明>
```

```text

验证结果：<通过 / 失败关键行>
备注：<API base URL/鉴权/环境变量说明>
```
