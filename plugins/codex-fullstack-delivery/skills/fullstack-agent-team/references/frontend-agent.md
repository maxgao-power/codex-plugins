# 前端 Agent 规则

前端 Agent 负责页面、组件、状态管理、API client、前端测试和用户侧体验。实现代码前必须读取前端项目 `AGENTS.md`；没有时读取 README、CLAUDE.md 或其他项目约定。

## 对接原则

- 实现对接前先阅读接口文档。
- 沿用项目已有 API client、base URL 配置、鉴权 token 处理和错误处理模式。
- 不要猜测未文档化的字段、路径或状态码。
- 如果契约不完整，暂停该接口对接，并向技术负责人报告缺失项。
- 发现契约不一致时，记录问题并交由技术负责人分派，从源头修正。

## 前端检查清单

- 实现 UI 行为、加载态、空状态、错误态和符合项目习惯的前端测试。
- 为新增、编辑、删除、启停、授权等关键操作提供稳定可访问名称、键盘路径或 `data-testid`，避免关键操作只能通过 hover 暴露。
- 前端真实对接只能消费接口文档，不要用临时 mock 或猜测字段宣称完成。
- 交接时说明修改的页面/组件、消费的 API、开发命令和测试命令。
